# 2. 规则

  • 「修饰符」
修饰符描述
i忽略大小写
g执行全局匹配 (查找所有匹配而非在找到第一个匹配后停止)
m执行多行匹配
s特殊字符圆点。中包含换行符 \n
  • 「表达式」
表达式描述
[abc]查找方括号之间的任何字符
[^abc]查找任何不在方括号之间的字符
[0-9]查找任何从 0-9 的数字
[a-z]查找任何从 a 到 z 的字符
[A-Z]查找任何从 A-Z 的字符
[A-z]查找任何从 A-z 的字符
[adgk]查找给定集合内的任何字符
[^adgk]查找给定集合外的任何字符
x|y匹配 x 或 y
(x)匹配 ‘x’ 并且记住匹配项，如 /(foo)(bar) \1 \2/ 中的 \1 、 \2 表示第一、二个被 捕获括号 匹配的子字符串
(?:x)匹配 ‘x’ 但不记住匹配项
x(?=y)匹配’x’仅仅当’x’后面跟着’y’。这种叫做先行断言。
(?<=y)x匹配’x’仅当’x’前面是’y’. 这种叫做后行断言。
x(?!y)仅仅当’x’后面不跟着’y’时匹配’x’，这被称为正向否定查找。
(?<!y)x仅仅当’x’前面不是’y’时匹配’x’，这被称为反向否定查找。
  • 「元字符」
元字符描述
.查找单个字符，除了换行和行结束符即 [^\r\n]（常见的匹配所有字符就是.*）
\w查找单词字符即 [0-9A-z_](包含下划线)
\W查找非单词字符即 [^\w]
\.*除换行符（\n、\r）之外的任何字符
\d查找数字
\D查找非数字字符
\s查找空白字符即 [\t\r\n\f\v](包含空格)
\S查找非空白字符
[\b]匹配一个退格键 (和换行符类似，是 \b)
\b匹配单词边界，如寻找单词 abc,/\babc\b/
\B匹配非单词边界
\0查找 null 字符
\n查找换行符寻找的是字符串中有无 "\n"
\f查找换页符
\r查找回车符
\t查找制表符
\v查找垂直制表符
\xxx查找以 8 进制数 xxx 规定的字符
\xdd查找以 16 进制数 dd 规定的字符
\uxxxx查找以十六进制数 xxxx 规定的 Unicode 字符
  • 「量词」
量词描述
n+匹配任何包含至少一个 n 的字符串 {1,}(贪婪匹配)
n*匹配任何包含零个或多个 n 的字符串
n?匹配任何包含零个或一个 n 的字符串 {0, 1}(非贪婪匹配)
n匹配包含 X 个 n 的字符串
n匹配包含 X 至 Y 个 n 的字符串
n匹配包含至少 X 个 n 的字符串
^n匹配任何开头为 n 字符串
n$匹配任何结尾为 n 的字符串
?=n匹配任何其后紧接指定字符串 n 的字符串 (正向预查)
?!n匹配任何其后没有紧接指定字符串 n 的字符串 (负向预查)<br /> ，其实也相当于 “排除 n”

# 3. 常用的正则表达式

正则表达式作用
^[0-9]*$纯数字
0-9^1[3|4|5|7|8]0-9$手机号码
^\d+$ 或 ^[1-9]\d*|0$非负整数
^[\u4e00-\u9fa5]{0,}$汉字
^\w+([-+.]\w+)*@\w+([-.]\w+)*.\w+([-.]\w+)*$email 地址
a-zA-Z0-9^(?=.\d)(?=.[a-z])(?=.*[A-Z])a-zA-Z0-9$强密码 (必须包含大小写字母和数字的组合，不能使用特殊字符，长度在 8-10 之间)
\n\s*\r空白行的正则表达式

# 4. 相关函数

# 4.1 字符串

string 相关作用
match(RegExp)查找匹配正则的文本的内容和位置
replace(RegExp, func)替换匹配的字符串
split(RegExp)根据正则切割原字符串
search(RegExp)查找第一个符合正则要求的文本出现的位置，没有则返回 - 1

# 4.2 RegExp 对象

RegExp 相关作用
test()查找字符串内是否存在匹配的文本，return: true/false
exec()修饰符为 g 时，每次调用返回匹配到的下一个文本，直至 null，然后再次调用则从头开始匹配；不为 g 时，返回第一个匹配到的文本，每次调用返回的结果是一样的

# 1. 思路

这里主要模拟一下菜鸟教程上的测试工具：

Notice：这里使用到了标签属性 "contentediable"，它可以支持对一些 DOM 直接键入文字，而且也可以用于实现富文本 (如 link 自动高亮、@提示等等)，并且也能自适应高度，因此也有替换 textarea 的做法。但是较为麻烦，要只实现纯文本键入，需要处理黏贴、拖拽事件（其实也有对应的纯文本键入的 css，但兼容性不好），还有光标位置等问题，更建议用现有的完成度高的富文本编辑器。

如前文所述，正则表达式主要可以看成两部分，字符 + 修饰符；

但经过测试，字符串难以转化为正确的字符，也即 <input/> 标签不好使，所以改用 contentediable 标签属性，获取 <pre> 标签内部的 textContent，最终得到正确的字符。

string -> RegExp 也有两种方法，一是 eval (不太安全)，二是 新建RegExp对象 的方式，但它们都会对特殊字符进行转义，不能正确转换。

获取到 字符 + 修饰符 后，通过 new RegExp 对象来构成正则表达式，随后利用 exec 方法找到对应的匹配文本和起始下标，对原数组进行分割，最后用 v-for 进行动态挂载，实现正则匹配的可视化。
